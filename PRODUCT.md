# Product

## Register

product

## Users

后端开发者和 SRE 工程师。他们需要在值班或排查告警时快速配置/修改 webhook 转发规则，预览消息渲染效果，并监控投递成功率。典型场景：凌晨收到告警，需要紧急调整消息模板或路由目标地址。

## Product Purpose

OpenHook 前端管理控制台，让开发者能够可视化地管理 webhook 转发服务的全部核心资源（模板、路由、中间件、令牌、投递日志），并实时预览消息渲染效果，无需手动调用 API。

## Brand Personality

精密、冷静、值得信赖。像手术刀一样精准，像仪表盘一样一目了然。

- **Precision**：每一个操作都有即时反馈，零猜测
- **Calm**：高密度信息但不压迫，重要状态一眼捕捉
- **Trustworthy**：投递状态、错误日志透明可见，没有黑箱

## Anti-references

- 不要像传统企业后台管理系统（臃肿、表格堆砌、左侧巨树导航）
- 不要像 Bootstrap Admin 模板（千篇一律的卡片+表单）
- 不要深色模式默认（虽然支持切换，但默认应为浅色，减少视觉疲劳）
- 不要渐变文字和玻璃拟态（装饰性过强，与工具属性冲突）

## Design Principles

1. **所见即所得**：模板编辑与实时预览并排，编辑后立即看到渲染结果
2. **零摩擦操作**：创建模板 → 创建路由 → 测试投递，三步内完成
3. **状态透明**：每次投递的请求/响应都完整记录，可追溯可调试
4. **键盘优先**：支持快捷键导航，让值班场景下的操作效率最大化
5. **防御性设计**：危险操作（删除）需要确认，关键配置变更前有预览

## Accessibility & Inclusion

- WCAG 2.1 AA 级别
- 支持键盘完全操作
- 支持 `prefers-reduced-motion`
- 颜色不单独承载语义（图标+文字双重编码状态）
